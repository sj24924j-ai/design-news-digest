# 要件定義書：デザイナー向け 毎朝ニュースダイジェスト（仮称）

- 作成日: 2026-07-20
- 対象ユーザー: 乃村工藝社デザイナー職（空間デザイン・ディスプレイ業界）
- 開発者: ユーザー本人（個人開発・非公開個人利用ツール）
- ステータス: ドラフト（壁打ちベース。「13. 未決事項」の確認後に実装着手）

---

## 1. 概要

乃村工藝社でデザイナーとして働くパートナーが、毎朝仕事に関連するニュースを効率よく確認できるWebアプリを作成する。空間デザイン・建築・展示会/万博・サステナブル素材など、業務に直結する分野のニュースを自動収集し、カテゴリ別に要約付きで一覧表示する。投資助言ツール（同ディレクトリの半導体関係性ツール）とは無関係の別プロジェクトとして、完全無料で運用できる構成を前提とする。

## 2. 背景・目的

- パートナーは空間デザイン・ディスプレイ業界の企業に勤務しており、業界動向・競合・トレンド素材・展示会情報などを日常的に追う必要がある。
- 現状は情報収集の手段が分散している、または手動で複数サイトを回る必要があり、毎朝の確認に時間がかかっている（想定）。
- 業務に関連するニュースだけを一箇所にまとめ、朝の身支度・通勤時間などの短時間で要点を把握できる状態を作ることが目的。
- 投資判断や意思決定ツールではなく、あくまで情報収集を効率化するための個人用ツール。

## 3. 想定ユーザー

- 主利用者: 乃村工藝社デザイナー職の1名（非公開・個人利用）。
- 開発・保守: ユーザー本人。ユーザー登録やアカウント管理は不要（単一ユーザー、認証機能なし）。

## 4. スコープ

### 4.1 対象ニュースジャンル（壁打ちで確定）

複数ジャンルを横断して収集し、カテゴリタブで切り替えられるようにする。

| カテゴリ | 内容例 |
|---|---|
| 空間デザイン・ディスプレイ | 商業施設・店舗・ミュージアム・展示演出、乃村工藝社および同業他社（丹青社、乃村工藝社グループ各社等）の動向 |
| 建築・インテリアデザイン | 建築家・インテリアデザイナー向けの一般的なデザイントレンド・作品事例 |
| イベント・展示会・万博 | 万博、MICE、展示会運営、大型イベントの企画・開催動向 |
| 素材・サステナブル | 内装・什器に使われる新素材、サステナブル建材、環境配慮デザインの動き |

### 4.2 スコープ外（本フェーズでは対応しない）

- 投資・株価等の金融情報（別プロジェクトである半導体関係性ツールの領域であり対象外）
- 複数ユーザー・ログイン機能
- プッシュ通知・メール配信（4.3参照、将来拡張候補）
- 記事本文の保存・全文表示（著作権配慮のため見出し・要約・リンクのみ）

### 4.3 確認スタイル

毎朝Webアプリ（ブラウザ）を開いて確認するダッシュボード形式。通知・メール配信は行わない（将来拡張候補として12章に記載）。

## 5. 機能要件

### 5.1 ニュース収集

ハイブリッド方式で収集する。

1. **固定ソース購読**: 業界メディア・企業サイトのRSSフィードを定期購読する（候補は9章）。
2. **キーワード検索による補完**: 固定ソースでカバーしきれないニュースを、キーワードベースの検索フィードで補完する。無料運用の方針に合わせ、LLMによる検索ではなく **Google Newsのキーワード検索RSS**（例: `https://news.google.com/rss/search?q=<キーワード>&hl=ja&gl=JP&ceid=JP:ja`、APIキー不要・無料）を利用する。
3. 収集は1日1回、深夜〜早朝のバッチ処理で実行し、朝には最新記事が反映された状態にする。

### 5.2 カテゴリ分類

- 各記事は取得元フィード or マッチしたキーワードに基づき、4.1のカテゴリのいずれか（複数可）に自動分類する。
- 分類ロジックはキーワードルールベース（フィードごとの固定カテゴリ + タイトル/本文中のキーワードマッチ）とし、LLMは使用しない（無料方針のため）。

### 5.3 要約・表示

- 要約は各RSSフィードが提供する`description`（見出し要約）をそのまま、または軽く整形して表示する。生成AIによる要約は行わない（無料方針のため。将来的な有料オプションとして12章に記載）。
- 記事本文はDB・ローカルに保存しない（出典サイトへのリンクのみ保持）。

### 5.4 画面構成

- **カテゴリ別タブ**: 「すべて」「空間デザイン・ディスプレイ」「建築・インテリア」「展示会・万博」「素材・サステナブル」の5タブ。
- 各タブ内は記事カードのリスト表示（新しい順）。カードには以下を表示:
  - タイトル（クリックで元記事に新規タブで遷移）
  - 出典名（メディア名）
  - 要約（RSS descriptionを2〜3行に整形）
  - 公開日時
  - サムネイル画像（RSSにあれば表示、なければ非表示）
- **日付ヘッダー**: 「本日 (7/20) 更新」のように、最終更新日時を明示する。
- **モバイルファースト**: 通勤中・身支度中などスマートフォンでの閲覧を主想定したレスポンシブデザインとする。

### 5.5 アーカイブ機能

- 過去の配信分を日付単位で見返せるようにする。
- 日付セレクタ（カレンダーUIまたはドロップダウン）で過去日を選択すると、その日に配信された記事一覧を表示する。
- データはバッチ実行のたびにJSONとして蓄積し、リポジトリ内に永続保存する（同一構成のリポジトリで運用している半導体関係性ツールのDBコミット方式を踏襲）。

## 6. 非機能要件

- **完全無料での運用**を必須要件とする。有料API・有料ホスティングは使用しない。
- 更新は平日毎朝（目安: 6:00 JST）までに完了していること。
- 個人利用のみを想定し、認証・アクセス制御機能は設けない。
- 元記事本文は保存せず、タイトル・出典・リンク・見出し要約のみを保持する（著作権配慮）。
- 想定記事数は1日あたり合計20〜40件程度（カテゴリ計）を目安とし、軽量な静的サイトとして高速に表示できること。

## 7. システム構成（アーキテクチャ）

半導体関係性ツール（本リポジトリ）と同じ「サーバーレス・完全無料」の構成方針を踏襲する。

```
[GitHub Actions（毎朝バッチ）]
   ├─ 固定RSSフィード取得
   ├─ Google News キーワード検索RSS取得
   ├─ カテゴリ分類（ルールベース）
   ├─ 重複排除・整形
   └─ 記事データをJSONとしてリポジトリにコミット（日付別に蓄積）

[GitHub Pages（静的サイト）]
   └─ フロントエンド（React + Vite等の静的ビルド）が
      リポジトリ内のJSONを読み込み、カテゴリタブ・アーカイブUIとして表示
```

- バックエンドサーバーは常設しない（GitHub Actionsのバッチ実行のみ）。
- フロントエンドは静的サイトとしてGitHub Pagesにデプロイする。
- 本ツールと同じリポジトリには含めず、**別リポジトリ・別プロジェクト**として新規作成する（技術方針の確認結果より）。

## 8. データ設計（記事JSONの例）

```json
{
  "id": "20260720-001",
  "title": "○○ミュージアムの新展示デザインが公開",
  "url": "https://example.com/news/xxxx",
  "source_name": "AXIS",
  "source_type": "fixed_rss",
  "categories": ["space_design"],
  "summary": "○○ミュージアムがリニューアルし、新しい空間演出を採用...",
  "thumbnail_url": null,
  "published_at": "2026-07-19T18:00:00+09:00",
  "fetched_at": "2026-07-20T06:00:00+09:00",
  "matched_keywords": []
}
```

- 日付ごとに `data/2026-07-20.json` のようなファイル名でアーカイブし、フロントエンドから日付指定で読み込む。

## 9. 情報源候補一覧（要確認・追加歓迎）

固定RSS購読の候補。実際の購読メディアはパートナー本人の希望も踏まえて9章末の未決事項で確定する。

| カテゴリ | サイト例 | 備考 |
|---|---|---|
| 空間デザイン・ディスプレイ | AXIS、JDN（Japan Design Net）、商店建築 | RSS提供有無を実装時に確認 |
| 建築・インテリア | Dezeen、designboom、10+1 website | 海外メディアは英語のまま表示 |
| 展示会・万博 | 大阪・関西万博公式ニュース、展示会関連業界メディア | RSS未提供の場合はスクレイピング要検討（本体ツールのTDnetスクレイピングと同様の運用リスクあり） |
| 素材・サステナブル | Sustainable Brands Japan 等 | 候補追加を要確認 |
| 企業公式 | 乃村工藝社プレスリリース、競合（丹青社等） | RSS有無を要確認。無ければキーワード検索側でカバー |

キーワード検索（Google News RSS）で補完する検索語の例:

- 「乃村工藝社」「空間デザイン」「展示デザイン」「ディスプレイ業界」「万博 パビリオン」「商業施設 デザイン」等

## 10. 開発・運用フロー

1. リポジトリ新規作成（GitHub、public想定）
2. GitHub Actionsで日次バッチ（RSS取得→分類→JSON生成→コミット）を実装
3. フロントエンド（React + Vite、カテゴリタブ・アーカイブUI）を実装しGitHub Pagesへデプロイ
4. 情報源リストの妥当性をパートナー本人と確認しながら調整（記事の的外れ・ノイズが多い場合はキーワード・購読先を見直す）

## 11. 制約・リスク・留意事項

- RSSを提供していないサイトはスクレイピングが必要になり、サイト構造変更に弱い（本体ツールの `tdnet_client.py` と同種の課題）。
- Google News RSSのキーワード検索は関連度が粗く、業務と無関係な記事が混ざる可能性がある。運用しながらキーワード・NGワードを調整する必要がある。
- 要約はRSS提供の見出し要約をそのまま使うため、フィードによっては情報量が少ない、または英語のままになる場合がある（AI要約を使わない無料方針とのトレードオフ）。
- GitHub PagesはリポジトリをPublicにする必要がある（無料プランの制約）。個人情報や非公開情報を含まない限り問題ないが、方針として明記しておく。

## 12. 将来の拡張候補（本フェーズはスコープ外）

- Claude API等を使った高品質な日本語要約・翻訳（有料オプション。半導体関係性ツールにおける「デフォルト無料・LLM抽出のみ有料オプション」という設計思想を踏襲する形）

- メール／LINE等への自動配信
